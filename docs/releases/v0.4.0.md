# r3f-interactive-flow v0.4.0

v0.4.0 was the input, transition, and test hardening release. These notes summarize the implemented and test-backed behavior that shipped in `r3f-interactive-flow@0.4.0`.

## Summary

- Hardens core navigation guard coverage for accepted navigation, rejected navigation, active transitions, locks, cooldowns, boundaries, same-phase requests, and invalid targets.
- Adds focused lifecycle coverage for wheel, touch, and keyboard input hooks.
- Verifies public type exports remain intentional.
- Synchronizes README behavior guidance with the tested core and input behavior.
- Added v0.4.0 release-readiness checklist documentation.

## Core behavior hardening

- Accepted `next`, `prev`, and `goTo` navigation starts transitions predictably and updates cooldown state according to the documented flow behavior.
- Rejected navigation does not partially mutate transition, lock, or cooldown state.
- Active transition, lock, cooldown, boundary, same-phase, and invalid-target guards are covered by regression tests.
- Cooldown guard coverage verifies that accepted navigation starts cooldown behavior and ignored requests do not reset or extend cooldown unexpectedly.

## Input hook lifecycle coverage

- Wheel input tests cover listener attachment, cleanup on unmount, disabled state, retarget cleanup, ignore selectors, threshold handling, axis handling, and hook-local cooldown behavior.
- Touch input tests cover listener attachment, cleanup on unmount, disabled state, retarget cleanup, threshold handling, axis handling, and hook-local cooldown behavior.
- Keyboard input tests cover listener attachment, cleanup on unmount, disabled state, retarget cleanup, repeat handling, typing target guards, key mapping behavior, and hook-local cooldown behavior.

## Public API and types

- Public type export coverage verifies the package root exports the intended public types.
- v0.4.0 did not add a new public runtime API.
- Runtime exports remain centered on `FlowProvider`, `useFlow`, `useFlowProgress`, `useFlowFrame`, `useWheelInput`, `useTouchInput`, and `useKeyboardInput`.

## Documentation

- README behavior guidance has been synchronized with tested navigation guard, cooldown, locking, and input hook lifecycle behavior.
- The v0.4.0 roadmap includes a release-readiness checklist for maintainers to use before publishing.
- These notes are historical release notes for the published v0.4.0 package.

## Scope and non-goals

The v0.4.0 hardening scope did not include:

- Release automation changes.
- New runtime dependencies.
- Public runtime API expansion.
- Visual effects, particle APIs, camera preset APIs, shader APIs, animation timeline APIs, router integration, GSAP integration, or Framer Motion integration.

## Maintainer validation used for release review

For v0.4.0 release review, maintainers verified or were expected to verify:

- `pnpm release:check` passes from the repository root.
- Package output includes `dist/`, `README.md`, `LICENSE`, `CHANGELOG.md`, and `package.json`.
- Built package outputs preserve the required `"use client"` directive.
- The release notes, changelog entry, and package version are reviewed together.
- No unrelated files, release automation changes, tags, GitHub Releases, or publish commands were included in the release-prep PR.
