# r3f-interactive-flow v0.5.0

v0.5.0 is a narrow documentation and example stabilization release. It prepares the package metadata and release notes for `r3f-interactive-flow@0.5.0` without publishing the package from this PR.

## Summary

- Aligns the root README public API and export documentation with the package root exports.
- Synchronizes package README validation guidance with the root README validation matrix.
- Adds README coverage for the Vite basic example.
- Clarifies Next.js Client Component boundaries for provider, hook, and browser input usage.
- Documents validation command guidance by PR type.
- Updates roadmap progress for the completed v0.5.0 documentation and example stabilization pass.

## Documentation and example stabilization

- Root documentation now keeps the current package status, release notes link, roadmap link, public runtime primitives, and public type exports aligned.
- Package-facing documentation follows the same validation guidance used by the root README so maintainers and contributors can choose checks by PR type.
- The Vite basic example has README guidance for running and building the example from the workspace.
- Next.js guidance explicitly treats components that use `FlowProvider`, `useFlow`, `useFlowProgress`, `useFlowFrame`, or browser input hooks as Client Components.
- Validation guidance distinguishes documentation-only, example, runtime/type, package-output, and release-prep review checks.

## Scope and non-goals

This v0.5.0 release-prep work does not include:

- Runtime code changes.
- Public runtime API expansion.
- New runtime dependencies.
- Visual effects, camera presets, shader APIs, animation timelines, router integration, GSAP integration, Framer Motion integration, or large templates.
- Release automation changes.
- npm publishing.
- Git tag creation.
- GitHub Release creation.

Publishing, tag creation, and GitHub Release creation are separate maintainer actions after release-prep validation.

## Maintainer validation

Before publishing, maintainers should run the non-publishing release readiness check from the repository root:

```bash
pnpm release:check
```

`pnpm release:check` is expected to chain build, package verification, typecheck, tests, lint, formatting, Vite example build, and package dry-run checks. It does not publish, create tags, or create GitHub Releases.

For the full maintainer checklist, see [../release.md](../release.md).
