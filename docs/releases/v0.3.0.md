# r3f-interactive-flow v0.3.0

v0.3.0 expands the phase, input, and frame-update foundation for predictable React Three Fiber interactive websites. It adds more flexible browser input hooks, typed frame callback state, and a preferred transition configuration API while keeping the package focused on control flow rather than visual effects or release automation.

## Summary

- Adds scoped wheel and touch input options for direct element/window targets, axis selection, ignore selectors, and hook-local cooldowns.
- Improves keyboard input configuration with grouped next/previous keys, typing guards, direct target support, and hook-local cooldowns.
- Updates `useFlowFrame` to pass a typed frame state object to Canvas-bound callbacks.
- Introduces `transition` and `transition.byPhase` options for global and source-phase transition timing.
- Refreshes README guidance and the Vite basic example around DOM UI controls, Canvas frame updates, input hooks, transition options, and browser safety.

## Added

- Scoped wheel input options, including direct element/window targets, axis selection, ignore selectors, and hook-local cooldown.
- Improved touch input options, including direct element/window targets, axis selection, ignore selectors, and hook-local cooldown.
- Improved keyboard input guards with grouped `keys.next` / `keys.prev`, configurable typing guards, direct target support, and hook-local cooldown.
- `FlowFrameState` and `FlowFrameCallback` public types.
- Typed `useFlowFrame((state, delta) => ...)` callback payloads.
- `transition` options for preferred global transition timing.
- `transition.byPhase` options for source-phase-specific transition timing.
- Public transition option types: `FlowTransitionBaseOptions` and `FlowTransitionOptions`.

## Changed

- `useFlowFrame` callbacks now receive `(state, delta)` instead of `(progress, delta)`.
- `transition` is now the preferred timing API.
- Legacy timing props remain supported: `transitionDurationMs`, `cooldownMs`, and `easing`.
- When both APIs are used, `transition` takes precedence over legacy timing props per field.
- The Vite basic example now demonstrates the current DOM UI to Canvas usage pattern.
- README documentation now covers DOM React UI controls, Canvas-bound frame updates, input hooks, transition options, migration notes, and Next.js/browser safety.

## Migration notes for `useFlowFrame` callback

`useFlowFrame` now passes a frame state object as the first callback argument instead of passing the raw progress number directly.

Before:

```ts
useFlowFrame((progress, delta) => {
  // Use progress directly.
});
```

After:

```ts
useFlowFrame(({ progress }, delta) => {
  // Destructure progress from the frame state object.
});
```

The new object payload makes the frame callback easier to extend while preserving the normalized progress value. Existing callbacks that only need progress should destructure `progress` from the first argument.

## Compatibility notes

- `nextKeys` and `prevKeys` remain supported as deprecated compatibility aliases for keyboard input.
- `keys.next` and `keys.prev` are the preferred keyboard configuration API.
- `transitionDurationMs`, `cooldownMs`, and `easing` remain supported for compatibility.
- `transition` is the preferred timing API for new code.
- `createFlowMachine` remains internal and is not exported from the package root.
- Current intended public exports include `FlowProvider`, `useFlow`, `useFlowProgress`, `useFlowFrame`, `useWheelInput`, `useTouchInput`, `useKeyboardInput`, `FlowFrameState`, `FlowFrameCallback`, `FlowInputTarget`, `FlowTransitionBaseOptions`, `FlowTransitionOptions`, `UseWheelInputOptions`, `UseTouchInputOptions`, and `UseKeyboardInputOptions`.
- `react`, `react-dom`, `three`, and `@react-three/fiber` are peer dependencies and should be installed by the application.

## Scope / non-goals

This release does not include the following:

- Package version changes in this documentation draft.
- Tags, GitHub Releases, npm publishing, or release automation.
- New runtime dependencies.
- Visual effect APIs.
- Camera preset APIs.
- Shader effect APIs.
- Animation timeline APIs.
- Router integration.
- GSAP or Framer Motion integration.
- Loop, queue, restart, interrupt, retarget, public source-phase, public target-phase, or `lockDuringTransition` behavior.
