# r3f-interactive-flow v0.6.0

v0.6.0 is a narrow input stabilization release. The release-prep PR prepared the package metadata and release notes for `r3f-interactive-flow@0.6.0`; the package has since been published to npm, and the GitHub Release for v0.6.0 has been created.

## Summary

- Documents the v0.6.0 roadmap and planning baseline for narrow input stabilization work.
- Validates wheel input threshold behavior around configured boundary values.
- Validates touch input threshold behavior around configured boundary values.
- Keeps threshold validation guidance synchronized in package documentation.
- Adds hook-local cooldown guard tests for ignored or blocked wheel input events.
- Adds hook-local cooldown guard tests for ignored or blocked touch input events.
- Adds hook-local cooldown guard tests for ignored or blocked keyboard input events.

## Input stabilization

The v0.6.0 stabilization work stays focused on existing wheel, touch, and keyboard input behavior. It hardens threshold validation and cooldown guard coverage without adding public options, changing package boundaries, or introducing new integrations.

Wheel and touch threshold validation is documented and covered around boundary values so applications can rely on predictable accepted and ignored input behavior. Hook-local cooldown tests now cover ignored or blocked wheel, touch, and keyboard events to protect the intended guard behavior from regressions.

## Scope and non-goals

The v0.6.0 release-prep PR did not include:

- npm publishing.
- Git tag creation.
- GitHub Release creation.
- Release automation changes.
- Dependency additions.
- Public API expansion.
- Unrelated runtime expansion.
- Visual effects, camera presets, shader APIs, animation timelines, router integration, GSAP integration, Framer Motion integration, or large templates.

Publishing, tag creation, and GitHub Release creation remain separate maintainer actions after release-prep validation. For v0.6.0, the npm publish and GitHub Release are complete.

## Maintainer validation

Before publishing a future release, maintainers should run the non-publishing release readiness check from the repository root:

```bash
pnpm release:check
```

`pnpm release:check` is expected to chain build, package verification, typecheck, tests, lint, formatting, Vite example build, and package dry-run checks. It does not publish, create tags, or create GitHub Releases.

For the full maintainer checklist, see [../release.md](../release.md).
